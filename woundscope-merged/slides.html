WoundScope

Reads messy nursing-home records and flags which wound patients can be billed to Medicare Part B — with the reasoning and evidence behind every call.

Team — Aadit · Aishwarya · Jay · Om● live demo
The problem

Billable wound care is hiding in the paperwork.

To bill Medicare Part B for a wound, three things must be true — an active wound, active Part B coverage, and proper documentation (size + drainage).

  • That evidence is scattered across coverage records, diagnosis codes, and messy free-text nurse notes.
  • Today a biller reads every patient by hand — slow, error-prone, and billable care gets missed.
What we built

One tool that reads it all and triages every patient.

WoundScope pulls the records, extracts the wound details, and routes each patient into one of three piles — each decision explained and backed by the source text.

Auto-Accept

Clean — ready to bill.

Flag for Review

Eligible, but a human should check.

Reject

Not Part B billable.

How it works

An automatic assembly line for the data.

→

→

3 · Decide

→

4 · Show

A dashboard with the decision, the proof, and a biller workflow.

The hard parts we solved

Real-world data is messy. We handled it.

  • A 30% failing connection → resilient ingestion with retries; zero records lost.
  • Notes in every format → parsers for the real templates, not the docs' fake example.
  • Garbled text → repairs like "Rightlowerle" → "right lower leg".
  • Two wounds in one note → detected and billed separately, never silently missed.
  • Full diagnosis coverage → pressure, diabetic, venous, arterial, burns, surgical, abscess.
Why a biller can trust it

Every decision shows its work.

  • Evaluation log — each rule as pass / fail.
